<CardGroup cols={2}>
  <Card title="Simple configuration" icon="list-radio">
    Manage everything from props, categories, permissions, keybinds, limits, notifications, all in one place
  </Card>

  <Card title="Prop categories" icon="folder">
    Organise props into categories, for example props for construction, or classrooms
  </Card>

  <Card title="Advanced placement" icon="mouse-pointer">
    Ghost previous, prop rotation and customisable grid snapping, plenty of options are available including undo and delete all
  </Card>

  <Card title="Edit mode" icon="edit">
    Dedicated system for editing props, allowing users to click any prop they own to move, delete or copy it. Authorised users can also edit any prop
  </Card>

  <Card title="Permissions" icon="user-group-simple">
    Each category can be locked to specific group ranks or in-game teams, restricting access to certain props
  </Card>

  <Card title="Admin override" icon="person">
    Add configurable role IDs or individual user IDs that can bypass limits, cooldowns, restrictions
  </Card>

  <Card title="Prop limits" icon="arrows-maximize">
    Players can have a configurable maximum number of props available to place, as well as a server prop cap
  </Card>

  <Card title="Mobile support" icon="mobile">
    Dedicated built in support with adjustments for anyone using a mobile device
  </Card>

  <Card title="Mappable keybinds" icon="keyboard">
    Fully fledged customisation for mapping keybinds to actions however you want, for example changing Cancel Placement
  </Card>

  <Card title="Customisable props" icon="traffic-cone">
    Ability to easily insert your own assets and props with ease, even without configuring them individually
  </Card>

  <Card title="Asset packs included" icon="boxes-packing">
    With Placeables, you will be given exclusive items and assets from our range of packs
  </Card>

  <Card title="No placement zones" icon="circle-stop">
    Ability to implement restricted area or zones that prohibit placement of any prop within it
  </Card>

  <Card title="Persistent prop saving" icon="database">
    Optional DataStore saving if you'd like props that have been placed to save continuously
  </Card>
</CardGroup>
